Стандартный цикл работы:

git clone -b nuzhnaya_vetka https://github.com/GolosChain/golos
cd golos
... правим файлы ...
git add *
git commit -m "This is commit header"
git push origin nuzhnaya_vetka
rm -rf golos

(!) Будьте внимательны с командой git push, именно она всегда загружает все на сервер гитхаба. Недаром именно она требует логин и пароль.

(!) Всегда помните, что если ваша ветка долго стоит, то в основной ветке появляются коммиты, которых нет в вашей. Лучше в таком случае вовремя удалять ветку, копировать ее заново из основной, качать к себе и с ней работать. Это предотвратит проблемы при пулл-реквестах.

=========================================

Удалить ветку old_branch:

git clone https://github.com/GolosChain/golos
cd golos
git push origin :old_branch
cd ..
rm -rf golos

========================================

Скопировать ветку golos-v0.18.0 в new_branch:

git clone https://github.com/GolosChain/golos
cd golos
git checkout golos-v0.18.0
git checkout -b new_branch golos-v0.18.0
git push origin new_branch  
cd ..
rm -rf golos

Или: в браузере зайти в ветку для копирования, слева (Branch: xxx) нажать мышкой, в поиск ввести имя new_branch (имя новой ветки) и нажать Create branch.

========================================

Переименовать ветку new_banch в new_branch:

git clone https://github.com/GolosChain/golos
cd golos
git checkout new_banch
git branch -m new_banch new_branch
git push origin :new_banch
git push --set-upstream origin new_branch 
cd ..
rm -rf golos 

========================================

Если нужно изменить заголовок коммита, но, к счастью, git push с этим коммитом еще не сделали:

Тогда делаем так:
git commit --amend
Открывается редактор, меняем заголовок, сохраняем и выходим.
Затем все как обычно:
git push origin nuzhnaya_vetka
cd ..
rm -rf golos

========================================

Если нужно изменить заголовок коммита или еще что-то в истории, которая УЖЕ на гитхабе:

Делаем:

git clone -b nuzhnaya_vetka https://github.com/GolosChain/golos
cd golos

Посмотреть историю коммитов (не в гитхабе, а в данном репозитории) можно так
git log
Вам надо определить сколько последних коммитов ребейзить. Допустим 3.
(если после этого не можете вернуться в командную строку, выполните команду из одной буквы q)

А чтобы править историю, нужно так: (последние 3 коммита в данном случае - помните, что все они будут обновлены, поэтому это число не должно включать в себя ни одного старого коммита, а то они тоже пойдут в пулл)
git rebase -i HEAD~3
Теперь правим в редакторе.
Если нужно слить коммиты в один, то тот коммит, который нужно оставить, не трогаем, а у следующих за ним меняем pick на fixup. После чего сохраняем и закрываем.
Если нужно отредактировать заголовок коммита из истории, то меняем на reword. После чего сохраняем, закрываем, оно снова открывается - и тут мы редактируем заголовок, и опять сохраняем и закрываем. Всё.

Если все прошло успешно, то проверяем:
git log

И затем пушим:
git push -f origin HEAD
cd ..
rm -rf golos

Если же с rebase не получается, тогда самое простое решение - удалить старую ветку (НЕ ЗАБЫВ ПРО БЭКАП), скопировать новую, заменить нужные файлы (только их) на бэкапленные и коммитнуть все как надо. Как это сделать - см. выше.

(i) Созданный пулл-реквест автоматически обновляется при ребейзе ветки.

(!) А вот git filter-branch вообще не используйте, чтобы править историю, потому что меняет SHA1 всем коммитам, и не получается сделать пулл-реквест.

(!) Если вы ребейзом затронули лишние (старые) коммиты и они теперь попали в пулл, то следует заново сделать rebase, поставить тем коммитам drop вместо pick и все сохранить и запушить.

========================================

(!) Не используйте Revert в пулл-реквестах.

(i) Еще раз: если с веткой слишком много проблем, то проще удалить старую ветку (НЕ ЗАБЫВ ПРО БЭКАП), скопировать новую, заменить нужные файлы (только их) на бэкапленные и коммитнуть все как надо. Как это сделать - см. выше.
Но открытый пулл в этом случае закроется и придется создавать его заново.
